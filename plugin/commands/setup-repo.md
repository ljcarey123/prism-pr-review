# Setup Repo — Enable prism GitHub integration

Adds the prism Actions workflow to the current repo and walks through GitHub Pages setup so PR report links are automatically posted on every PR.

---

## Step 1 — Check current state

Run these in parallel:
```bash
git rev-parse --show-toplevel        # confirm we're in a git repo
git remote get-url origin            # get the GitHub repo URL
gh auth status                       # confirm gh CLI is available
```

If not in a git repo, stop: "Run this from inside the repo you want to enable prism on."
If `gh` is not available, note the user will need to complete the Pages step manually.

---

## Step 2 — Check if workflow already exists

Check whether `.github/workflows/pr-report.yml` exists in the repo root.

- If it already exists, tell the user and ask if they want to overwrite it. If no, skip to Step 4.

---

## Step 3 — Copy the workflow file

Locate the prism workflow source. It's at the path returned by:
```bash
npm root -g 2>/dev/null && echo "global" || echo "local"
```

Or find it via npx:
```bash
node -e "require.resolve('prism-pr-review')" 2>/dev/null
```

If that fails (package not globally installed), look for it relative to the Claude commands directory — the workflow can also be embedded directly.

Write the following workflow file to `.github/workflows/pr-report.yml`:

```yaml
name: Deploy PR Review Report

on:
  pull_request:
    types: [opened, synchronize, reopened]

permissions:
  contents: write       # push to gh-pages branch
  pull-requests: write  # post PR comments

jobs:
  deploy-report:
    runs-on: ubuntu-latest
    steps:
      - name: Checkout PR branch
        uses: actions/checkout@v4
        with:
          fetch-depth: 0   # need full history to diff base..head

      - name: Check report is up-to-date for this PR
        id: check
        env:
          BASE_SHA: ${{ github.event.pull_request.base.sha }}
          HEAD_SHA: ${{ github.event.pull_request.head.sha }}
        run: |
          if git diff --name-only "$BASE_SHA" "$HEAD_SHA" | grep -q "^\.pr/index\.html$"; then
            echo "fresh=true" >> $GITHUB_OUTPUT
          else
            echo "fresh=false" >> $GITHUB_OUTPUT
          fi

      - name: Deploy to GitHub Pages
        if: steps.check.outputs.fresh == 'true'
        uses: peaceiris/actions-gh-pages@v4
        with:
          github_token: ${{ secrets.GITHUB_TOKEN }}
          publish_dir: .pr
          destination_dir: prs/${{ github.event.pull_request.number }}
          keep_files: true

      - name: Post or update PR comment — report ready
        if: steps.check.outputs.fresh == 'true'
        uses: actions/github-script@v7
        with:
          script: |
            const owner  = context.repo.owner;
            const repo   = context.repo.repo;
            const prNum  = context.issue.number;
            const sha    = context.payload.pull_request.head.sha.substring(0, 7);
            const head   = context.payload.pull_request.head.ref;
            const base   = context.payload.pull_request.base.ref;
            const url    = `https://${owner}.github.io/${repo}/prs/${prNum}/`;

            const body = `## 🔮 PR Review Report\n\n**[View interactive report →](${url})**\n\n| | |\n|---|---|\n| Branch | \`${head}\` |\n| Target | \`${base}\` |\n| Commit | \`${sha}\` |\n\n*Generated by [prism](https://github.com/ljcarey123/prism-pr-review) 🔮*`;

            const comments = await github.rest.issues.listComments({ owner, repo, issue_number: prNum });
            const existing = comments.data.find(c => c.body.includes('🔮 PR Review Report'));

            if (existing) {
              await github.rest.issues.updateComment({ owner, repo, comment_id: existing.id, body });
            } else {
              await github.rest.issues.createComment({ owner, repo, issue_number: prNum, body });
            }

      - name: Post or update PR comment — report missing
        if: steps.check.outputs.fresh == 'false'
        uses: actions/github-script@v7
        with:
          script: |
            const owner  = context.repo.owner;
            const repo   = context.repo.repo;
            const prNum  = context.issue.number;

            const body = `## 🔮 PR Review Report\n\n> **No up-to-date report found for this PR.**\n\nTo generate one, run \`/ship\` in [Claude Code](https://claude.ai/code) before pushing:\n\n\`\`\`\n/ship\n\`\`\`\n\nThis will analyse the diff, generate an interactive report at \`.pr/index.html\`, and publish it here automatically.\n\n---\n> ⚠️ **GitHub Actions must be enabled** on this repository for auto-deployment to work.\n> If you don't see this comment update after pushing, check: **Settings → Actions → General → Allow all actions**.\n\n*[prism](https://github.com/ljcarey123/prism-pr-review) 🔮*`;

            const comments = await github.rest.issues.listComments({ owner, repo, issue_number: prNum });
            const existing = comments.data.find(c => c.body.includes('🔮 PR Review Report'));

            if (existing) {
              await github.rest.issues.updateComment({ owner, repo, comment_id: existing.id, body });
            } else {
              await github.rest.issues.createComment({ owner, repo, issue_number: prNum, body });
            }
```

After writing:
```bash
git add .github/workflows/pr-report.yml
git commit -m "chore: add prism PR review workflow"
git push
```

---

## Step 4 — Check GitHub Actions is enabled

```bash
gh api repos/{owner}/{repo}/actions/permissions 2>/dev/null
```

Parse the response:
- If `enabled: true` — Actions is on, all good.
- If `enabled: false` or error — tell the user:
  > GitHub Actions appears to be disabled on this repo.
  > Enable it at: **Settings → Actions → General → Allow all actions**

---

## Step 5 — Bootstrap GitHub Pages

The workflow publishes to a `gh-pages` branch, but GitHub Pages must be enabled manually in the repo settings. The `gh-pages` branch won't exist until the first workflow run.

Tell the user:

> **Pages setup (one-time, after your first PR)**
>
> 1. Run `/ship` on a feature branch — this triggers the workflow and creates the `gh-pages` branch
> 2. Once the Actions run completes, go to:
>    **Settings → Pages → Source: Deploy from branch → gh-pages → / (root) → Save**
> 3. All future PRs will automatically get an interactive report link in the PR comments

If `gh` is available, check whether Pages is already enabled:
```bash
gh api repos/{owner}/{repo}/pages 2>/dev/null
```
- If it returns a `url` field — Pages is already live, tell the user.
- If it 404s — Pages not yet set up, show the instructions above.

---

## Step 6 — Done

Summarise what was done:
- Whether the workflow was added/already present
- Whether Actions is enabled
- Pages status and what to do next

Remind the user: **run `/ship` on your next feature branch** to generate the first report.
